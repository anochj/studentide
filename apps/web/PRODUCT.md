# Product

## Register

product

## Users
studentide serves instructors and students. Instructors create coding project definitions, configure environments, launch or manage IDE sessions, and review submissions. Students discover assigned or public projects, work in a prepared development environment, and submit completed work.

## Product Purpose
studentide is a web app for creating, distributing, running, and reviewing coding projects. It should make project setup feel structured and credible, reduce friction around development environments, and give both instructors and students a clear path from project definition to completed submission.

## Brand Personality
Technical, focused, capable. The product should feel like a serious builder tool with enough distinctiveness to be memorable in a resume portfolio, without looking decorative, generic, or visibly AI-generated.

## Anti-references
Avoid generic SaaS dashboards, stock developer-tool tropes, template-like card grids, purple-gradient AI styling, overused "modern app" layouts, and interfaces that feel assembled from defaults without product judgment.

## Design Principles
1. Make the workflow legible: creation, discovery, IDE sessions, and submissions should each have a clear role.
2. Show technical credibility through precision, not visual noise.
3. Design for repeated use: instructors and students should be able to scan, compare, and act quickly.
4. Keep the portfolio signal strong: the product should demonstrate taste, restraint, and craft.
5. Prefer specific product language over generic platform copy.

## Accessibility & Inclusion
Target WCAG AA. Preserve keyboard access, visible focus states, sufficient contrast, reduced-motion support, and layouts that remain usable for both instructor-heavy dashboard workflows and student-facing project views.
